# Зарница 2.0 — бесконечный тренажёр запоминания

Статический сайт для GitHub Pages. Работает без сервера и без установки программ.

## Механика тренировки

- На экране всегда один вопрос.
- Следующий вопрос выбирается случайно.
- Вопрос исчезает из тренировки только после **двух правильных ответов**.
- Если ученик ошибся, он видит правильный ответ и пояснение, а вопрос позже появится снова.
- Тренировка продолжается, пока все выбранные вопросы не будут закреплены.

## Что есть

- Режим ученика с ником.
- Выбор блока вопросов.
- Статистика для скриншота: закреплено, осталось, попытки, верно, ошибки, процент, время.
- Админ-панель.
- Редактирование вопросов, вариантов, правильных ответов и пояснений.
- Добавление картинок по ссылке или загрузкой файла в админке.
- Смена пароля администратора.
- Просмотр статистики игроков в админке.
- Экспорт вопросов и статистики в JSON/CSV.

## Как разместить на GitHub Pages

1. Создайте публичный репозиторий на GitHub.
2. Распакуйте архив.
3. Загрузите **содержимое папки**, а не сам zip.
4. В репозитории откройте `Settings → Pages`.
5. Выберите `Deploy from branch`, ветка `main`, папка `/ root`.
6. Через минуту сайт будет доступен по адресу вида:
   `https://ваш-логин.github.io/название-репозитория/`

## Про картинки

Лучший вариант для GitHub:

1. Положить картинку в папку `assets`.
2. В вопросе указать путь, например: `assets/q81.jpg`.

Также можно вставить внешнюю ссылку вида `https://...`.

Загрузка файла через админку тоже работает, но такая картинка сохраняется в браузере как встроенные данные. Для постоянного сайта надёжнее хранить картинки в папке `assets` на GitHub.

## Важное ограничение GitHub Pages

GitHub Pages — это статический сайт. У него нет общей базы данных.

Поэтому статистика в админке видна только та, которая сохранена в текущем браузере. Для сбора результатов со всех учеников используйте скриншоты статистики или экспорт JSON/CSV. Для общей онлайн-таблицы нужна версия с Firebase/Supabase.
